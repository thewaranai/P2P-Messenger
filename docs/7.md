# Диаграмма классов

На основе кода приложения (упрощённо) предлагается следующая структура:

```mermaid
classDiagram
    class User {
        -int id
        -String username
        -String password
        -String status
        +getUsername()
        +setStatus()
    }

    class Message {
        -int id
        -int senderId
        -int receiverId
        -String content
        -Date timestamp
        -boolean isDelivered
        +getContent()
    }

    class Contact {
        -int userId
        -int contactId
        -Message lastMessage
    }

    class ConnectionManager {
        -Socket socket
        -String peerIp
        -int port
        +connect(ip, port)
        +disconnect()
        +sendMessage(Message)
        +receiveMessage() : Message
    }

    class ChatActivity {
        -ListView messageList
        -EditText inputField
        -Button sendButton
        -ConnectionManager connection
        +displayMessage(Message)
        +sendMessage()
    }

    class LoginActivity {
        -EditText username
        -EditText password
        -Button loginButton
        +authenticate()
    }

    class DatabaseHelper {
        +saveUser(User)
        +getUser(username) : User
        +saveMessage(Message)
        +getHistory(contactId) : List~Message~
    }

    User "1" --> "0..*" Message : отправляет
    User "1" --> "0..*" Message : получает
    User "1" --> "0..*" Contact : имеет
    ChatActivity --> ConnectionManager
    ChatActivity --> DatabaseHelper
    LoginActivity --> DatabaseHelper
    ConnectionManager --> Message
